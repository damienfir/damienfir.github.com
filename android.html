Android

All the informations below are tested with a Samsung Galaxy S II and should be valid with most of the other Android-based phone out there.

Rooting and ROMs

The general structure of an Android phone is arranged the following way:

  • The kernel which is used to interface with the hardware. You can flash (replace) the kernel with a modified one which is rooted (allows the applications to used superuser privileges)
  • The ROM which the user uses to interface with the kernel. This is the software that can be changed to replace the stock ROM and add features to the Android OS.

I recommend going to http://forum.xda-developers.com/forumdisplay.php?f=1055 to find kernels and ROMs.

APK installation

The adb utility is useful for interfacing with the phone directly from a computer. It is used to copy file to/from the phone and install applications (.apk) remotely.

Installing an application involves the following command: apk install ***.apk

If it is a system application, you need to push (copy) the entire tree application to the phone: apk push system/ system/
